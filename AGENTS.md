# AGENTS.md

## このリポジトリの目的
- このリポジトリは、競技プログラミング用の個人 Python ライブラリである。
- 実装は、再利用しやすく、貼り付けやすく、実戦で使いやすいことを重視する。
- 一般的な業務ソフトウェアのような過剰な抽象化・過剰な防御コードは避ける。

## 実行環境と依存
- Python コードは PyPy3 での利用を前提とする。
- 外部依存は原則として追加しない。標準ライブラリを優先する。
- 小さな補助関数のために大きな依存や複雑な構成を導入しない。

## 実装方針
- 各アルゴリズムファイルは、できるだけ自己完結に保つ。
- 小さい用途に対して、過度なクラス分割やフレームワーク的設計を持ち込まない。
- 競技プログラミング用途として、定数倍・簡潔さ・使い回しやすさを重視する。
- まず正しさと計算量を優先し、そのうえで定数倍を意識する。
- ホットパスでは不要なオブジェクト生成を避ける。

## コードスタイル
- 既存のリポジトリの書き方に合わせる。
- 関数ベースの実装を基本とする。
- 型注釈は、可読性や使用法の理解に役立つ範囲で付ける。
- コメントや docstring は日本語で簡潔に書く。
- 公開関数や主要データ構造には、役割・引数・返り値・計算量・不変条件が分かる説明を優先する。
- 競プロ用途なので過剰なフォールバック処理は不要。
- ただし、軽い境界値チェックや明らかな misuse を防ぐ簡単なチェックは許容する。

## verifier / verify-helper
- verify-helper コメントは必須とする。
- verify 用ファイルを新規追加・更新するときは、必要な verify-helper コメントを必ず付ける。
- 既存の verify-helper コメントや verifier 用メタデータは壊さない。
- 検証対象のコードの挙動を変えた場合は、対応する verify ファイルやドキュメントへの影響を意識する。
- verifier 管理下の生成物や補助ファイルは、明確な理由がない限り不用意に編集しない。

## 変更時の注意
- 既存 API の名前・意味・計算量・使用感をなるべく保つ。
- 非自明な変更では、どの不変条件を保ったか、何が変わったかを説明する。
- 計算量が変わる場合は明記する。
- 既存の利用箇所や verify に影響が出る変更は避け、必要なら最小限に留める。

## ディレクトリ方針
- `python/`: ライブラリ本体。競プロでの再利用性と verifier との整合性を優先する。
- `docs/`: 実コードと一致する説明を書く。説明だけ先行して更新しない。
- `.github/workflows/`: CI・verify・docs まわり。明示的に求められたとき以外は触らない。
- verifier 管理ディレクトリや生成物は、ツール管理物として扱う。

## Codex への作業ルール
- 小変更では、既存スタイルに寄せた最小差分を優先する。
- 要求されていない大規模リファクタリングは行わない。
- 単なる見た目の統一より、正しさ・verify 互換性・再利用性を優先する。
- 新規コード追加時は、そのファイル単体で読んで使いやすい形を優先する。
